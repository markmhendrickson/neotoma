# Release

Canonical release command. Use `/release`.

## Purpose

Create or prepare a release with the full release workflow, including:

- Release planning
- Full release-detail preview and confirmation
- Dependency analysis and execution scheduling
- npm and GitHub release preparation, including unmerged `dev` work

## Command

Follow `foundation/agent_instructions/cursor_commands/create_release.md` in full.

## Notes

- `/release` is the canonical command name.
- `/create_release` is a legacy alias.
- Before writing release files, preview **all release details** and get explicit confirmation.
