## اپنے ایجنٹ کو فیصلہ کرنے دیں کہ کیا نیوٹوما فٹ بیٹھتا ہے۔

ہم ایک تشخیصی صفحہ فراہم کرتے ہیں جو آپ کا AI ایجنٹ پڑھ سکتا ہے۔ ایک مضبوط ایجنٹ کو آپ کے ٹول اور ورک فلو کے سیاق و سباق کا معائنہ کرنا چاہیے، ایک یا دو ٹارگٹڈ سوالات پوچھنا چاہیے، اور پھر آپ کو صاف صاف بتانا چاہیے کہ کیا نیوٹوما فٹ بیٹھتا ہے۔

اس پرامپٹ کو اس میں کاپی کریں۔ کوئی بھی AI ایجنٹ تشخیصی صفحہ پڑھنے کے لیے، اپنے ٹول، ورک اسپیس، اور کنفیگریشن سیاق و سباق کا معائنہ کریں، پھر فیصلہ کریں کہ کیا Neotoma آپ کے حقیقی ورک فلو پر فٹ بیٹھتا ہے اور پہلے کیا برقرار رکھنا ہے۔

## اکثر پوچھے گئے سوالات

پلیٹ فارم میموری (کلاڈ، چیٹ جی پی ٹی) کافی اچھی ہے - ایک اور ٹول کیوں شامل کریں؟

پلیٹ فارم میموری اسٹور کرتا ہے جسے ایک وینڈر یاد رکھنے کا فیصلہ کرتا ہے، اس فارمیٹ میں آپ معائنہ یا ایکسپورٹ نہیں کرسکتے ہیں۔ یہ ورژن نہیں بناتا، تنازعات کا پتہ نہیں لگاتا، اور اگر آپ ٹولز کو تبدیل کرتے ہیں تو غائب ہو جاتا ہے۔ Neotoma آپ کو منظم، کراس ٹول حالت دیتا ہے جسے آپ کنٹرول کرتے ہیں۔

کیا میں اسے صرف SQLite یا JSON فائل سے نہیں بنا سکتا؟

آپ وہاں شروع کر سکتے ہیں - بہت سی ٹیمیں کرتی ہیں۔ لیکن آپ کو آخرکار ورژن سازی، تنازعات کا پتہ لگانے، اسکیما ارتقاء، اور کراس ٹول مطابقت پذیری کی ضرورت ہوگی۔ یہ انفراسٹرکچر کا کام مہینوں کا ہے۔ Neotoma ان ضمانتوں کو پہلے دن بھیجتا ہے۔

کیا یہ پیداوار کے لیے تیار ہے؟

Neotoma ڈویلپر کے پیش نظارہ میں ہے - اصلی ایجنٹ ورک فلوز کے ذریعہ روزانہ استعمال کیا جاتا ہے۔ بنیادی ضمانتیں (تعمیراتی حالت، ورژن کی تاریخ، صرف ضمیمہ لاگ) مستحکم ہیں۔ 5 منٹ میں انسٹال کریں اور اپنے ایجنٹ کو فٹ کا اندازہ کرنے دیں۔

کیا Neotoma میرا ڈیٹا کلاؤڈ کو بھیجتا ہے؟

نہیں، Neotoma مقامی طور پر بطور ڈیفالٹ چلتا ہے۔ آپ کا ڈیٹا مقامی SQLite ڈیٹا بیس میں آپ کی مشین پر رہتا ہے۔ یہاں کوئی کلاؤڈ سنک، کوئی ٹیلی میٹری، اور آپ کے ڈیٹا پر کوئی تربیت نہیں ہے جب تک کہ آپ API کو ظاہر کرنے کا انتخاب نہیں کرتے ہیں (مثال کے طور پر ریموٹ MCP کلائنٹس کے لیے)۔

RAG میموری اور deterministic میموری میں کیا فرق ہے؟

RAG متن کے ٹکڑوں کو اسٹور کرتا ہے اور اشارے کے لیے مماثلت کے ذریعے بازیافت کرتا ہے۔ نیوٹوما ساختی مشاہدات کو اسٹور کرتا ہے اور کم کرنے والوں کے ساتھ ہستی کی حالت کو مرتب کرتا ہے۔ ایک جیسے مشاہدات ہمیشہ ایک ہی سنیپ شاٹ حاصل کرتے ہیں۔ RAG مطابقت کو بہتر بناتا ہے۔ ڈیٹرمنسٹک میموری سالمیت، ورژن سازی، اور آڈٹ ایبلٹی کو بہتر بناتی ہے۔
