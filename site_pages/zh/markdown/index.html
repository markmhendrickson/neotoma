## 常见问题

平台内存（Claude、ChatGPT）已经足够好了 - 为什么要添加另一个工具？

平台内存以一种您无法检查或导出的格式存储供应商决定记住的内容。它不进行版本控制，不检测冲突，并且在您切换工具时消失。 Neotoma 为您提供可控制的结构化、跨工具内存。

我不能只使用 SQLite 或 JSON 文件构建它吗？

你可以从这里开始——很多团队都是这样做的。但您最终将需要版本控制、冲突检测、模式演变和跨工具同步。这是数月的基础设施工作。 Neotoma 在第一天就提供了这些保证。

这个产品准备好了吗？

Neotoma 处于开发者预览版 — 真实代理工作流程每天使用。核心保证（确定性内存、版本化历史记录、仅附加更改日志）是稳定的。 5 分钟内完成安装，并让您的代理商评估是否合适。

Neotoma 会取代 Claude 的记忆或 ChatGPT 的记忆吗？

不——它与他们一起工作。平台内存存储供应商决定在该供应商的工具中记住的内容。 Neotoma 存储您在所有工具中控制的事实。继续使用平台内存来快速了解上下文；当您需要版本控制、可审核性和跨工具一致性时，请使用 Neotoma。

Neotoma 会将我的数据发送到云端吗？

不会。Neotoma 默认在本地运行。您的数据保留在您计算机上的本地 SQLite 数据库中。除非您选择公开 API（例如对于远程 MCP 客户端），否则不会进行云同步、遥测和数据训练。

RAG 内存和确定性内存有什么区别？

RAG 存储文本块并通过相似性检索它们。 Neotoma 存储结构化事实并为每个事实建立版本化历史记录；相同的输入总是产生相同的结果。 RAG优化相关性；确定性内存可优化完整性、版本控制和可审核性。

记忆力会随着时间的推移而退化或漂移吗？

不会。Neotoma 使用带有确定性缩减器的仅附加观察日志。没有任何内容被覆盖或默默删除。六个月前存储的事实与今天存储的事实一样可检索和验证 - 完整的版本历史记录和出处完好无损。记忆化合物；它永远不会腐烂。
