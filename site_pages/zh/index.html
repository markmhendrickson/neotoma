常见问题

平台内存（Claude、ChatGPT）已经足够好了 - 为什么要添加另一个工具？

平台内存以一种您无法检查或导出的格式存储供应商决定记住的内容。它不进行版本控制，不检测冲突，并且在您切换工具时消失。 Neotoma 为您提供结构化的、跨工具的控制状态。

我不能只使用 SQLite 或 JSON 文件构建它吗？

你可以从这里开始——很多团队都是这样做的。但您最终将需要版本控制、冲突检测、模式演变和跨工具同步。这是数月的基础设施工作。 Neotoma 在第一天就提供了这些保证。

这个产品准备好了吗？

Neotoma 处于开发者预览版 - 真实代理工作流程每天使用。核心保证（确定性状态、版本化历史记录、仅附加日志）是稳定的。 5 分钟内完成安装，并让您的代理商评估是否合适。

Does Neotoma send my data to the cloud?

No. Neotoma runs locally by default.您的数据保留在您计算机上的本地 SQLite 数据库中。除非您选择公开 API（例如对于远程 MCP 客户端），否则不会进行云同步、遥测和数据训练。

RAG 内存和确定性内存有什么区别？

RAG 存储文本块并通过提示的相似性检索它们。 Neotoma 存储结构化的观察结果并与减速器组成实体状态；相同的观察总是会产生相同的快照。 RAG优化相关性；确定性内存可优化完整性、版本控制和可审核性。
