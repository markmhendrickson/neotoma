Neotoma with Claude · Local setup for Claude Desktop (stdio transport).

Setup

Paste this prompt into another agent tool (for example Claude Code or Cursor) to install Neotoma and configure it for Claude Desktop. The agent handles npm install, initialization, and MCP configuration.

Follow the canonical Neotoma onboarding sequence: 1. Start at https://neotoma.io/evaluate and assess whether Neotoma fits my workflow. 2. Determine what data would be highest value to store first and whether I am already installed. 3. If Neotoma is a fit and I am not installed, run npm install -g neotoma and neotoma init. Report what changed. 4. If I am already installed, skip re-installation unless something is broken. 5. Activate Neotoma: ask which data types matter most to me (project files, chat transcripts, meeting notes, notes/journals, code context, email, financial docs, or custom paths). Offer three modes: quick win, guided, or power user. 6. Discover high-value local files: scan shallowly based on my preferences. Rank by entity density, temporal signals, and recency. Group results into domains, not file counts. Also gather candidate data from available context and platform memory. 7. Explain why each recommendation was selected and what timeline/entity value it could unlock. 8. Present one preview with provenance. Let me confirm per-folder or per-file. Show expected reconstruction before ingesting. Pause and wait for my approval. 9. Ingest confirmed files. Reconstruct the strongest timeline with provenance: each event traced to a specific source file. Show the timeline immediately -- not a file count. 10. Offer one follow-up query specific to the reconstructed timeline. Then suggest 2-4 next actions based on my data. Ask if the reconstruction is accurate and support corrections. 11. After activation, configure the tool I am using for the most robust ongoing Neotoma workflow (for example: Custom GPT or remote MCP for ChatGPT when supported, Projects plus connector setup for Claude, local MCP wiring first for Cursor / Claude Code / Codex / OpenClaw). Canonical source: https://github.com/markmhendrickson/neotoma/blob/main/install.md

Claude Desktop uses local stdio. Neotoma runs on the same machine. No API server or remote access is required. The agent writes to ~/Library/Application Support/Claude/claude_desktop_config.json and you restart Claude Desktop to pick up the new MCP server.
