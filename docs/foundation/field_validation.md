# Field Validation and Market Evidence

## Build-in-house explosion

Developers are independently building their own agent memory systems. This is the strongest market timing signal: the problem is real enough that 10+ teams have built their own solutions, confirming both the need and the timing.

**Named evaluator attributions:** See `docs/private/icp/field_evidence.md`.

### What this validates

1. **Market timing is correct.** The problem is acute enough that builders invest significant engineering time in custom solutions rather than accepting the status quo.
2. **The homebrew solutions share common weaknesses.** No versioning, no entity resolution, no provenance, no cross-tool access, no schema evolution. Each builder re-discovers these limitations independently.
3. **The ICP is building infrastructure for themselves, not shipping products.** These are personal tools, not startups. The personal agentic OS builder archetype is validated.

## Agent-vs-human evaluation divergence

A consistent pattern across evaluations: agents with tool-use capabilities assess Neotoma more favorably than the same person's human assessment.

### Divergence patterns

| Type | Agent assessment | Human assessment | Interpretation |
|------|-----------------|------------------|----------------|
| **Tool-use agents** (MCP, function calling) | Enthusiastic, specific use cases identified | Generally positive but less specific | Agents with infrastructural agency recognize the state layer need from experience |
| **Search-only agents** (web-based evaluation) | Skeptical, "solution looking for a problem" | Varies | Agents without tool experience lack the context to evaluate a state layer |
| **Platform builder** (builds own infra) | "Neotoma-shaped problem" | "I don't have the problem" | Starkest divergence — agent sees architectural need, human builds their own |
| **Thought-partner user** (human-in-the-loop) | Self-selected out: "context and voice, not deterministic state" | Agreed | Clean alignment — both agent and human correctly identified non-ICP |

### Implications

- AEO (agent evaluation optimization — Neotoma's usage; distinct from industry "answer engine optimization", see `docs/foundation/product_positioning.md` §7.5) is a real acquisition funnel — agents are making or influencing adoption decisions
- The site should be optimized for agent legibility (structured data, verifiable claims, machine-readable comparisons)
- Agents with tool access are the best evaluators because they already need state management
