## Material Changes Definition

Material changes require user notification (30 days' notice) and may require re-acceptance:

- Changes to payment terms or pricing
- Changes to service availability or features
- Changes to user obligations or prohibited activities
- Changes to limitation of liability or warranties
- Changes to dispute resolution procedures
- Changes to termination or cancellation policies

Non-material changes (notification not required):

- Clarifications or formatting improvements
- Updates to contact information
- Corrections of typos or errors
- Updates to service description (if no functional change)

---

## Re-Acceptance Requirements

**Re-acceptance required for:**

- Material changes to payment terms
- Material changes to limitation of liability
- Material changes to dispute resolution
- Changes that reduce user rights

**Re-acceptance not required for:**

- Clarifications or non-material updates
- Updates that expand user rights
- Corrections or formatting changes

---

## Compliance Notes

- **Notification:** Material changes require 30 days' notice before taking effect
- **Re-acceptance:** Users must re-accept Terms if material changes affect their rights
- **Documentation:** All changes must be documented in this changelog
- **Version Control:** Track changes in git (markdown source)
- **Continued Use:** Continued use after notification constitutes acceptance (if re-acceptance not required)
